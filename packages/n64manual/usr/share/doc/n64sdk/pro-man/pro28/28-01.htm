28.1 Flash ROM

A cartridge containing 1 Mbit Flash ROM (1M Flash) has been made available as a Nintendo64 back-up device to accommodate the increasing size of back-up data.

The 1M Flash is accessed as a PI device, the same way as SRAMs have been accessed traditionally. It differs from SRAMs in the following ways.

Differences in Writing Methods
One of the flash ROM characteristics of 1M Flash is that data must be erased before data can be written. Data are erased in one-sector units (16 Kbytes).

After the data have been erased, the data are transferred temporarily into a 128-byte write buffer which is inside the flash ROM. Then they are actually written. For this reason, data are written in one-page units (128 bytes).

Differences in Reading Methods
Depending upon the flash ROM version, it may be necessary to modify the data reading method of 1M Flash.

Please use the flash read function available in the library to avoid this problem.

Differences in Size
The size has been enlarged to 1 Mbit.

Note: Please do not use 0x3fe and 0x3ff, which have been set aside for future use by Nintendo. (Sector erasures including pages 0x3fe and 0x3ff are not a problem.)

Differences in Speed
The speed parameter of 1M Flash as a PI device (PI device access speed) is the same as an SRAM. However, there will be some extra processing that accompanies the actual writing and reading operations, which take place in one-page units. For this reason, the speed (especially when writing) can be much slower.
