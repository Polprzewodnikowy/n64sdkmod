４．４エラーメッセージ

【エラー】
起動時 "OLE initialization failed. Make sure that the OLE libraries are the correct version."
「OLE の初期化に失敗しました。OLE ﾗｲﾌﾞﾗﾘが正しいﾊﾞｰｼﾞｮﾝのものか確認してください。」
アプリケーション起動時のOLEのDLLを初期化に失敗すると発生します。 一度、 アプリケーションをUninstallしてInstallし直してみて下さい。
"SolidPalette.ocx is not found."
SolidPalette.ocx がアプリケーションのあるディレクトリ(c:\nintendo\n64kit\texedit\bin)か、 WindowsのSystemディレクトリに見つからない時に発生します。 このファイルはアプリケーションが必要とするモジュールファイルです。 一度、 アプリケーションをUninstallしてInstallし直してみて下さい。
"BrowseFolder.ocx is not found"
BrowseFolder.ocx がアプリケーションのあるディレクトリ(c:\nintendo\n64kit\texedit\bin)か、 WindowsのSystemディレクトリに見つからない時に発生します。 このファイルはアプリケーションが必要とするモジュールファイルです。 一度、 アプリケーションをUninstallしてInstallし直してみて下さい。
"PropertyList.ocx is not found"
PropertyList.ocx がアプリケーションのあるディレクトリ(c:\nintendo\n64kit\texedit\bin)か、 WindowsのSystemディレクトリに見つからない時に発生します。 このファイルはアプリケーションが必要とするモジュールファイルです。 一度、 アプリケーションをUninstallしてInstallし直してみて下さい。
TPFファイル操作時 "In File, Keyword was not found."
「ファイルにキーワードが見つかりませんでした。」
TPFファイルを開く時にTPFとして重要なキーワード(TPF_DIR/TMF_DIR/TLF_DIR)が見つからなかった時に発生します。 TPFファイルとして正しくないか、TPFファイルでない可能性があります。 正しいTPFファイルを開いて下さい。
"キーワード - "ディレクトリ" is not exist directory."
TPFファイルの重要なキーワード(TPF_DIR/TMF_DIR/TLF_DIR)の設定したディレクトリパスが存在しない時に発生します。 正しいディレクトリ・パスを設定して下さい。
TMFファイル操作時 "compressed format not supported."
圧縮BMPファイルを開こうとした時に発生します。 非圧縮BMPファイルに変換してから開き直して下さい。
"Sorry, 24bit not supported. 4bit or 8bit only support."
1bit,16bit,24bit,32bitのBMPファイルを開こうとした時に発生します。 4bit,8bitのBMPファイルに変換してから開き直して下さい。
"save failed."
TMFファイルの保存に失敗した時に発生します。 このメッセージの前に、 システムエラーメッセージが出力されます。 システムエラーメッセージに応じて対処して下さい。 おそらく、パス名が正しくないか、 共有ロックが掛かっているか、 読み込み専用ファイルになっているか、 ディスク容量が足りないか、 ディスクが挿入されていないか、 ハードウェアのトラブルだと思われます。
TLFファイル操作時 "[File] file open failed."
TLFファイルの読み込み・書き込みに失敗した時に発生します。 正しいTLFファイル名を指定して下さい。
"[File] incorrect body size."
"[File] create failed."
"[File] incorrect header information."
"[File] incorrect header size."
TLFファイルの読み込みに失敗した時に発生します。 TLFファイルじゃないか、TLFファイルが壊れている可能性があります。 正しいTLFファイルを指定して下さい。
ピクチャー操作時 "Please specify new file name."
「新しいﾌｧｲﾙ名を指定して下さい。」
ピクチャーダイアログで、「新規作成(New...)」ボタンを押した時に、 コンボボックスにファイル名が入力されていない場合に発生します。 新規TMF／TLFを作成するためには、 名前が必要ですので重複しない(UNIQUEな)名前を入力して下さい。
"File name has already existed. Please give unique name."
「ﾌｧｲﾙ名が衝突しています。他の名前に変えて下さい。」
ピクチャーダイアログで、「新規作成(New...)」ボタンを押した時に、 コンボボックスに既に存在するファイル名が入力されている場合に発生します。 新規TMF／TLFを作成する際は、 重複しないUNIQな名前を入力して下さい。
"Incorrect parameter."
「ﾊﾟﾗﾒｰﾀｰが正しくありません。」
新規プロジェクトを作成する時に、 入力パラメータが足りない場合に発生します。 おそらく、 プロジェクト名を指定していない事が原因だと考えられますので、 プロジェクト名を指定して下さい。
"An active palette not found. You must open or create a palette."
イメージをプロジェクトのピクチャに登録する時に、同時にパレットがアタッチしようとしたが、 アタッチできるパレットがない場合に発生します。 アタッチしたいパレットを開くか、作成して下さい。
プラグイン操作時 "Preview Plug-In not installed."
"preview entry not found"
プレビュー用のプラグインが起動時にPlugInsディレクトリに無い状態で、プレビューボタンを押した時に発生します。 アプリケーションのあるディレクトリから、 "PlugIns/Preview.*" が、あるかどうか確認して下さい。 無ければ、Installし直して下さい。
"Import Plug-In not installed."
"import entry not found"
"Export Plug-In not installed."
"export entry not found"
インポート／エクスポート用のプラグインが起動時にPlugInsディレクトリに無い状態で、インポート／エクスポートを行おうとした時に発生します。 ヘルプメニューのバージョン情報(英語版："About N64 CITextureEditor...")を開いて、インポート／エクスポート用のプラグインが インストールされているか確認して下さい。 無ければ、 インストーラーからプラグインを再Installし直して下さい。
"ISVNLoad.dll is not found. Please, Install the IS-VIEWER64 software."
プレビューに必要なISVNLoad.dllが見つからない場合に発生します。 プラグインディレクトリにISVNLoad.dllが無ければ再インストールして下さい。 もしくは、IS-VIEWER64のソフトウェアをインストールして下さい。 WindowsのSystemディレクトリにあれば問題ありません。
"available IS-VIEWER64 is not found. You can connect to max 7 IS-VIEWER64 on one SCSI host adapter."
接続可能なIS-VIEWER64が見つからない場合に発生します。 他のプログラムによってIS-VIEWER64が使用中のため、接続できない場合があります。 同時利用を避けて下さい。
ヘルプ呼び出し時 "can not open help. Please, check exist help file, and check browser installed."
「ﾍﾙﾌﾟが表示できません。 ﾍﾙﾌﾟのﾌｧｲﾙが存在するか、 またはﾌﾞﾗｳｻﾞが正しくｲﾝｽﾄｰﾙされているかを確認してください。」
ヘルプファイルが表示できない時に発生します。 アプリケーションのあるディレクトリから、 "../doc/index.html" を探しますので、 パスが存在するか確認して下さい。 それでも表示されない場合は、 拡張子HTMLからブラウザを起動する様にアソシエーション(Association)されていないか、 ブラウザが正しくインストールされていない可能性があります。 ブラウザを再度インストールし直して下さい。
【警告】
プロジェクト操作時 "TPF file already exists. Over write?"
「TPF ﾌｧｲﾙが既に存在しています。 上書きしますか？」
新規プロジェクトを作成する時に、指定した名前のファイルが既に存在する場合、警告されます。 そのまま上書きするなら「はい(YES)」を、取り消すなら「いいえ(NO)」を選択して下さい。
"Project already opened. (You can use only one Project.) Do you close it?"
「既にﾌﾟﾛｼﾞｪｸﾄは開かれています。 このﾌﾟﾛｼﾞｪｸﾄを閉じてから開きますか？」
新規プロジェクトを作成する時や、プロジェクトを開く時に、既にプロジェクトが開いていると発生します。 プロジェクトは１つしか開けないので、現在編集中のプロジェクトを閉じて新しいプロジェクトを開く場合は「OK」を、 そのままの状態に戻る場合は「キャンセル(Cancel)」を押して下さい。
ピクチャー操作時 "file is not found. Do you ignore error to continue?"
「ファイル が見つかりません。 無視して継続しますか？」
ピクチャーダイアログでＯＫボタンを押した時に、指定したTMF及びTLFファイルが見つからない時に発生します。 おそらく、ファイル名が正しくないか、TMFディレクトリ、TLFディレクトリの指定に誤りがあるかだと考えられます。 無視して継続するなら「OK」を、ファイル名を修正するなら「キャンセル(Cancel)」を選択して下さい。
"You must attach a palette before add to project. attach to active palette now?"
イメージをプロジェクトのピクチャに登録する時に、パレットがアタッチしていないか、 アタッチしていたパレットが有効でない場合に発生します。 現在アクティブなパレットをアタッチするなら「はい(YES)」を選んで下さい。 キャンセルする場合は、「いいえ(NO)」を選んで下さい。
"Please save document that adds to project."
イメージをプロジェクトのピクチャに登録する時に、イメージがファイルとして存在しない場合に発生します。 この警告ダイアログの後に、ファイル保存ダイアログが開きますので、ファイルとして保存して下さい。
イメージ操作時 "Could not allocate buffer for UNDO information. Do you clear all UNDO buffer?"
イメージ操作時にUNDOバッファを確保するメモリが足りなくなると発生します。 UNDOバッファをクリアする場合は「はい(YES)」を選んで下さい。「いいえ(NO)」を 選んだ場合は、出来れば不要なウィンドウを一度閉じてメモリを確保して下さい。
