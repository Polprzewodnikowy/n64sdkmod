N64 CITextureEditor TPF Format


English

  1. TPFとは

      テクスチャ・イメージ・ファイル (TMF)と、 テクスチャ・ルックアップテーブル・ファイル (TLF)を 一対に登録した複数の「ピクチャー」を管理するファイルです。
      N64 CITextureEditorでは１つのTPFのみ編集可能です。

      Makefile風のテキストファイルで表現されています。

      例）<sample.tpf> 
      #TPFのあるディレクトリ
TPF_DIR = C:\nintendo\n64kit\texedit\user_data
#TMFのあるディレクトリ
TMF_DIR = $(TPF_DIR)
#TLFのあるディレクトリ
TLF_DIR = $(TPF_DIR)
#登録されたピクチャーの数
PICT_NUM = 2
#ピクチャー（PICT.<n> = <ピクチャー名>,<TMF>,<TLF>,<palette>）
PICT.0 = Picture0, Texture0.tmf, Palette0.tlf, 0
PICT.1 = Picture1, Texture1.tmf, Palette0.tlf, 0

  2. 書式

      基本的な書式のルールは

      1. #<コメント>
      2. <マクロ名> = <値（文字列）>
      3. 値（文字列）に$(マクロ名)で他のマクロを参照する事が出来る。
      です。

      TPFで予約されたマクロ名は以下の通りです。

      1. TPF_DIR … TPFの格納したディレクトリを示す (パスの区切りは"/"も利用可能)
      2. TMF_DIR … TMFの格納したディレクトリを示す (パスの区切りは"/"も利用可能)
      3. TLF_DIR … TLFの格納したディレクトリを示す (パスの区切りは"/"も利用可能)
      4. PICT_NUM … 登録されたピクチャーの数(0〜n)
      5. PICT.<n> … ピクチャの情報(<n>=0〜PICT_NUM-1)
          PICT.<n> = <ピクチャー名>,<TMF>,<TLF>,<palette>
