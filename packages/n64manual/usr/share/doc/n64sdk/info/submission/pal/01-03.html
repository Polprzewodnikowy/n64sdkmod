N64 Software Specification Sheet Instructions

To ensure accurate information and reduce delays in processing your software submission, we now provide the N64 Software Specification Sheet as a PDF form. Complete and print this form, then include a copy of it with your software submission.

Instructions for Nintendo 64 Software Specification Sheet
The following paragraphs are intended to aid the licensee in completion of the Nintendo 64 Software Specification Sheet.

  • Game Title
    Enter the game's title, including any sub-titles.

  • Product Code
    Enter the game's Product Code (4 digits) as determined by Nintendo.

  • Language
    Enter the language which appears in the game as screen text and messages.

  • Accessories
    Indicate accessories other than standard Controller which can be used.

  • Audio Tools
    Indicate if M.O.R.T. (Advanced Sample Optimize Editor) is used for voice compression. All developers who incorporate M.O.R.T. voice compression into their game are required to make their best efforts to display a screen credit on the opening licensing screen, which states the following:
    "Voice Compression Technology Licensed by Factor 5 and LucasArts Entertainment Company"

    Also indicate whether MusyX Audio Tools were used in the game. All developers who incorporate MusyX Audio Tools into their game are required to make their best efforts to display a screen credit on the opening licensing screen, which states the following:

    "MusyX Audio Tools Licensed by Factor 5"

  • Disk Support
    Indicate whether the game supports disks. If so, enter the product name and abbreviation for the disk. (If no abbreviation has been set, enter "undecided.")

  • Expansion Pak
    If a Expansion Pak is absolutely essential for the game, check the "Essential" box on the form. If the Pak is supported but not essential, check the "Supported" box. Otherwise, check "Not Supported."

  • Overseas Version
    If the product has been sold, or is to be sold in another country; write the game title, country, product code, the scheduled release date in that country, and indicate if there are differences between the overseas and U.S. versions.

    In addition, please indicate if submitting master ROMs for both the Japanese and overseas versions at the same time, or if the overseas version has already been submitted and is currently being processed for approval.

    Indicate whether or not the submitted product has exactly the same ROM (data) as the overseas version, in the "Difference Between Japanese Specifications and Other.".

  • Contact Name
    Enter the name of the primary contact person for your software submission.

  • Company
    Enter your company's name.

  • Department
    Enter your department.

  • Address
    Enter your company's address, telephone number, and fax number.

  • Submission Date and Method
    Enter the date and method used for submission.

  • Scheduled Release Date
    Enter the planned release date for the software.

Game Title Registration
Write the contents registered in the indicated addresses of the Master ROM. Enter ASCII characters in the boxes above the dashed lines. Enter the 2-digit Character Code for each ASCII character below the dashed line. Please refer to the "Character Code List for Game Registration" for appropriate Character Codes.

Please also indicate the mask ROM version, entering the Version Number (0,1,2,3) in the upper box and its Character Code (00, 01, 02, 03) in the lower box.

Please refer to the "ROM Data Specification" for more information concerning ROM Registration.

Program Contents
Enter whether or not a Controller Pak is used. If a Controller Pak is used, indicate the number of Notes, the number of Pages, and the note name(s). The note name(s) should be related to the game title. List the Game Code and company code included in the Game Note.

Indicate the MakeMask software version number. The first MakeMask version is 2.02.

Indicate the version of the N64 Library Files and Debugger software used to create the program.

Memory Configuration

Enter the ROM size.

Back-up Memory:
If battery back-up is used within the Game Pak instead of using a Controller Pak, indicate the device used and its size.

Example
EEPROM: 4 Kbit, 16Kbit
SRAM: 256 Kbit

Game Boy Software for use with the N64 Transfer PakTM <
If the N64 Transfer Pak is used, please enter the name of the game company, the name of the software, and the product code. Please indicate whether the program allows the user to write to the N64 Transfer Pak.

ROM Version

  • Mask ROM Version: The Mask ROM Version number starts from 0 and increases for each revised version sent for changes after starting production.
  • Submission Version:
    The EEPROM Version number starts from 0 and increases for each revised version sent for approval.

Disk File Name, CRC

  • File Name
    Enter the file name and check sum of the data storage file using the following conventions.

  • CRC
    Enter the CRC as calculated using the NRDC software provided by Nintendo.
